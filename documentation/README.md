# Documentation Non-Code

Ce dossier centralise les documents non-code conservés dans le dépôt.

## Structure

- `technical/`: documents de cadrage, framework, templates, guides et specs.
- `legal/`: documents légaux et consentements.

## Règles de tri

- Garder uniquement les documents pertinents et informatifs pour le projet.
- Supprimer les artefacts personnels/opérationnels (ex. captures de billets d'avion).
- Supprimer les archives/ressources de rendu non essentielles (ex. ZIP legacy, images de fond de couverture).
- Éviter les doublons: conserver les variantes explicites si leur contenu diffère.
